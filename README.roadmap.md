## ü§ù Contributing

### Workflow
1. Fork the repo
2. Create a feature branch
3. Make changes (follow code style, add tests, update docs)
4. Test changes
5. Commit with Conventional Commits
6. Push and open a pull request

### Coding Standards
- **Backend (Java)**: Google Java Style, Lombok, 80% test coverage, Javadoc
- **Frontend (React)**: Functional components, Airbnb style, meaningful names, reusable components
- **Commits**: Conventional Commits (`type(scope): subject`)

### Project Structure
- Backend: `lego-factory-backend/<service-name>`
- Frontend: `lego-factory-frontend/src/{components,pages}`
- Config: project root
- Docs: update `.md` files as needed
